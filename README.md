# 📊 Customer Churn Prediction (Machine Learning)

This project explores a customer churn dataset from a telecommunications company, containing over 7,000 records. The objective is to identify customers at risk of leaving the service using supervised machine learning techniques.

---

## 🚀 Scope of the Project

This project focuses on the **core stages of a machine learning workflow**, including:

* Data cleaning and preprocessing
* Handling missing values and inconsistent data types
* Encoding categorical variables
* Train/test splitting with stratification
* Baseline model training (Logistic Regression)
* Model evaluation and metric interpretation
* Decision threshold tuning

---

## 📊 Results

* Achieved **~93% recall** on churn prediction after threshold tuning
* Precision dropped to ~43%, highlighting a trade-off between recall and false positives

This reflects a **recall-oriented modeling strategy**, prioritizing detection of churn cases over prediction precision.

---

## 🧠 Key Insight

The project demonstrates how model performance depends not only on the algorithm, but also on:

* Data preprocessing quality
* Handling of class imbalance
* Choice of evaluation metrics
* Decision threshold selection
